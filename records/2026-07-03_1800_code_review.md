### 已知 bug 修复（2026-07-03 18:00）

1. **parseCrs API key 解析 bug**：原版用 `s{h}s{a}` 生成 API key（h=a=0-5），但实际 API key 是 `s{HH}s{AA}` 2 位补 0（s00s00, s01s00, s05s05），导致所有 0 比分（0:0 / 0:1 / 1:0）被漏掉
   - 修法：用 `String(h).padStart(2,'0')` 生成 API key
   - 影响范围：data/odds/2040347/2040348/2040357.json 的 bf_latest 字段从 3 keys（仅 胜其它/平其它/负其它）扩展到 31 keys（含全部 0:0-5:5 比分）
2. **updateHistory first entry 丢失 bug**：原版 `[...historyArr.slice(0, -1), newEntry]` 在 historyArr.length=1 时会丢失 first entry
   - 修法：`[firstEntry, ...historyArr.slice(1, -1), newEntry]` 显式保留 first
